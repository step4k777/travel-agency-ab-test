## Описание проекта

Цель проекта — проанализировать результаты A/B-теста двух вариантов посадочной страницы туристического агентства и определить, какой из вариантов более эффективен по ключевым бизнес-метрикам.

В рамках исследования оценивались две метрики:

- конверсия в покупку;
- ежедневный средний чек.

Проект включает полный цикл анализа A/B-теста:

- предобработку данных;
- проверку корректности проведения теста;
- анализ ключевых метрик;
- анализ стабилизации метрик во времени;
- статистическую проверку гипотез;
- формулировку итоговой бизнес-рекомендации.

## Данные

Датасет содержит информацию о посещениях сайта туристического агентства и результатах взаимодействия пользователей с посадочной страницей.

Признаки:

- `user_id` — идентификатор пользователя;
- `date` — дата посещения сайта;
- `group` — группа теста (`A` — контрольная, `B` — тестовая);
- `purchase` — факт покупки тура:
  - `1` — покупка совершена;
  - `0` — покупка не совершена;
- `price` — стоимость купленного тура, если покупка состоялась, иначе `0`.

В рамках A/B-теста пользователям предлагались туры с фиксированной стоимостью:

- Таиланд — 100 000 рублей;
- Турция — 60 000 рублей;
- Мальдивы — 200 000 рублей;
- Санкт-Петербург — 10 000 рублей;
- Камчатка — 150 000 рублей.

Размер датасета: около 294 тыс. наблюдений.

## Этапы проекта

### 1. Загрузка и первичный анализ данных

- выполнено первичное знакомство с данными;
- проверены типы данных;
- исследована структура датасета.

### 2. Предобработка данных

- столбцы приведены к корректным типам данных;
- проверено отсутствие пропущенных значений;
- проверена длительность теста в группах A и B;
- удалены пользователи, попавшие одновременно в обе группы.

### 3. Первичный анализ результатов A/B-теста

Рассчитаны основные показатели по группам:

- количество визитов;
- количество покупок;
- суммарная выручка;
- конверсия;
- средний чек.

Также был выполнен анализ продаж по каждому туру в разрезе групп A и B.

### 4. Анализ стабилизации метрик

Для каждой группы были рассчитаны:

- ежедневное количество визитов;
- ежедневное количество покупок;
- ежедневная сумма покупок;
- ежедневная конверсия;
- ежедневный средний чек.

Дополнительно были построены кумулятивные метрики:

- кумулятивная конверсия;
- кумулятивный средний чек.

### 5. Статистический анализ

Для проверки различий между группами были использованы статистические тесты:

- z-test для сравнения конверсий;
- тест Шапиро–Уилка для проверки нормальности;
- статистический тест для сравнения ежедневного среднего чека между группами;
- построение 95% доверительных интервалов для конверсий, разницы конверсий и среднего чека.

## Результаты анализа

По итогам анализа было установлено:

- статистически значимого различия в конверсии между группами A и B не выявлено;
- доверительный интервал разницы конверсий включает 0, что подтверждает отсутствие значимого различия;
- ежедневный средний чек в группе B оказался выше, чем в группе A;
- доверительные интервалы среднего чека для групп не перекрываются, что указывает на значимое отличие.

Таким образом, вариант B показал лучший результат по метрике среднего чека при сопоставимой конверсии.

## Выводы

В рамках проекта был проведён анализ A/B-теста двух вариантов посадочной страницы туристического агентства.

По результатам исследования было установлено, что:

- вариант B не уступает варианту A по конверсии;
- вариант B обеспечивает более высокий средний чек;
- использование варианта B позволяет получать больший доход с покупки без снижения доли покупок.

**Итоговая рекомендация:** предпочтительным является вариант посадочной страницы **B**.
